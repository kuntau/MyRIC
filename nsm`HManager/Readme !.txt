INSTALLATION
—————————————————————————————————————————————————————————————————————————————————

1. First Extract the HManager.zip file into you main mirc directory. For example  C:\Program Files\Mirc
 
2. Next type /load -rs nsm`HManager\nsm`HManager.mrc on any mirc window.
 
3. The load dialog will popup and answer as you desire
 
4 To open "Hash Table Manager" you can type [ /HManager ] in any window or you can acces through your mirc menu bar

REVIEW
—————————————————————————————————————————————————————————————————————————————————

THE IDEA
One day while i'm writing my script i found it is very hard to manage large number of hash table especially when you left it for a few month. Yes, you can view all them
using command in mirc but it take time and need knowledge in mirc command. I'm thinking how i can manage my hash table easier and faster, and this is what i got

- General info about your hash table state on the right side table
- Feature status bar at the bottom
- Feature undo button so you can undo what you edit
- Undo is not available once you hit "proceed !"
- Feature some build-in error checker, some button will be disable if error occurred
- Error checker in switch, which mean you can only use switch that are available in mirc or else you can't proceed. Example : in /hmake command only switch 
  "snbi" available, if you include other switch an error message will appear at the status bar
- Error checker in hash table name to detect whether that hash table exists or not.
- Error checker in /hload file name whether it is exists or not.
- Check Ontop if you want the dialog on top.
